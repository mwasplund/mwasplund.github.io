## Opinions
It is important to strike a balance between heavy handed opinionated build systems and generic code execution engines masquerading as build systems. When you open a system that is turning complete programming language to write our build logic there is a large scope of functionality that could be represented. On the other hand, if you author the same definition using a declarative language there is only so much we can do. If we can live within these constraints then we can limit the complexity and allow ourselves to focus on more important tasks.

When I open a declaration file I know it has a single state of properties to reason over, I do not need to think through side effects of previous build states or look up the definitions of helper functions. What you see is what you get.

`Developers voluntarily impose constraints on the system in order to amplify their reasoning ability` -Just Enough Software Architecture
